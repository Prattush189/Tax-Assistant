## 2. Current project state

- **Framework:** React + TypeScript + Vite + Express + better-sqlite3
- **Dev server:** `npm run dev` — runs Vite on :3000 and Express (tsx watch) on :4001 via concurrently. WebSocket HMR works.
- **Build:** `npx vite build` → ~12s, ~2MB main bundle, no compile errors
- **DB:** `data/tax-assistant.db` (SQLite). Schema uses `IF NOT EXISTS` on startup; migrations in `server/db/index.ts` add columns to existing DBs.
- **RAG:** In-memory inverted index at startup. 7,081 chunks loaded from `comparison.txt`, `act-2025.txt`, `act-1961.txt`, `cgst-2017`, `igst-2017`, `reference-data.txt`, `gst_changes_2025.txt`. Disabled sources: SGST, amendments, finance acts.
- **AI:** Grok 4.1 Fast via OpenAI SDK pointing at `api.x.ai/v1`; Gemini 2.5 Flash Lite via OpenAI-compat endpoint.

### Critical server bugs fixed in this session
1. **`grok.ts` module-load crash** — `new OpenAI({apiKey: process.env.XAI_API_KEY})` was throwing synchronously when env was missing, crashing the entire Express process before routes were registered. Fix: placeholder key fallback + warning log.
2. **Schema migration order crash** — `schema.sql` had `CREATE INDEX ON users(external_id)` which ran before the `ALTER TABLE ADD COLUMN external_id` migration. Fix: removed all index creation from schema.sql; indexes are now created inside the ALTER block in `db/index.ts`.

Both fixes are in place — the server now starts cleanly with missing AI keys (chat/notices will fail cleanly at request time with a proper JSON error, but auth/calculator/notices-list all work).
