    - `fy`: '2025-26' | '2024-25' (default '2025-26')
    - `grossSalary`: string (controlled input, empty = 0)

    - `ageCategory`: AgeCategory (default 'below60')

    - `showDeductions`: boolean (toggle to show/hide old regime deduction fields)
    - `showHRA`: boolean

    Calculation (useMemo):
    - Parse all string inputs to numbers (empty string → 0)
    - Fetch rules = getTaxRules(fy)
    - Build IncomeTaxInput for old regime (with deductions + HRA) and new regime (deductions/HRA ignored)
    - Call calculateIncomeTax(oldInput, rules) and calculateIncomeTax(newInput, rules)
    - Return { oldResult, newResult } — recalculates on every input change

    Layout:
    - FY selector (radio buttons or select): "FY 2025-26" / "FY 2024-25"
    - Age category selector: "Below 60" / "Senior (60-80)" / "Super Senior (80+)"
    - Income fields: "Gross Annual Salary (₹)" and "Other Income (₹)"
    - Expandable section "Old Regime Deductions" (toggle showDeductions):
      - 80C (max ₹1.5L), 80D self, 80D parents, 80CCD(1B)
      - Checkboxes: "I am a senior citizen" (affects 80D self limit), "Parents are senior citizens"
    - Expandable "HRA Exemption" (toggle showHRA):
      - Actual HRA received, Basic + DA, Annual rent paid, Metro city toggle
    - Below inputs: render `<RegimeComparison>` passing oldResult, newResult, fy

    Input validation: Use HTML `type="number" min="0"` — no additional validation library. Show placeholder "0".

    IMPORTANT: Do NOT include calculation logic in this component. All math is in taxEngine.ts called via useMemo.
