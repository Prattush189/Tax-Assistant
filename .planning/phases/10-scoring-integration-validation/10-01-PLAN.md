Create server/rag/validate-golden.ts following the established validate-reference.ts pattern (import from './index.js', call initRAG(), use check() helper, exit 0/1).

The script must:

1. Read golden-queries.json using readFileSync + JSON.parse (not import assert — avoid ESM JSON import complexity; use the pattern: `const goldenQueries = JSON.parse(readFileSync(join(__dirname, '..', 'data', 'golden-queries.json'), 'utf-8'));`)

2. Call initRAG() to load all sources

3. **SCOR-01 confirmation check:** Verify DEFAULT_TOP_K by calling retrieveContextWithRefs("income tax", 5) and checking result.references.length === 5. Report as "[PASS/FAIL] SCOR-01: topK=5 returns 5 chunks"

4. For each golden query, call retrieveContextWithRefs(q.query, 5) and check:
   - **Source match:** At least one source from q.expectedSources appears in the returned references' source IDs. Report: "[PASS/FAIL] {q.id}: expected source present — sources={actual sources}"
   - **SCOR-02 label check:** Every returned reference has a non-empty .label that is NOT equal to the raw .source ID (labels should be human-readable like "IT Act 1961", not raw "act-1961"). Report: "[PASS/FAIL] {q.id}: source labels are human-readable"
   - **Value match (if expectedValues defined and non-empty):** For each expected value, at least one reference text includes that value string. Report: "[PASS/FAIL] {q.id}: expected value '{value}' found in results"

5. **Token budget measurement:** For each query, compute approximate tokens as Math.ceil(result.references.reduce((sum, r) => sum + r.text.length, 0) / 4). Track all values and report:
   - Average token count across 15 queries
   - Maximum (worst-case) token count
   - "[PASS/FAIL] Token budget: avg={avg} max={max} target=3000"
   - Pass if average <= 3000 (per CONTEXT.md target)
   - Warn (but still pass) if max > 3000 even when avg passes

6. Print summary: total checks, passed, failed. Exit 0 if all pass, exit 1 if any fail.

7. **Baseline diagnostic mode:** When RAG_DEBUG=1 env var is set, print per-query detail showing all 5 returned chunks with their source, section (first 50 chars), and score approximation (text.length). This helps diagnose scoring issues before Plan 02 tuning. The debug output is purely informational and does not affect pass/fail.
