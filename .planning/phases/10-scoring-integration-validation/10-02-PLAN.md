Modify server/rag/index.ts to fix the three confirmed retrieval problems. Changes are scoped entirely to SOURCE_CONFIGS and initRAG() — no changes to scoreChunk(), retrieve(), or any exported API.

**Step 1: Add `disabled` flag to SourceConfig interface:**
```typescript
interface SourceConfig {
  id: string;
  filePath: string;
  label: string;
  splitter: 'act' | 'comparison' | 'reference';
  boost?: number;
  disabled?: boolean;  // skip loading and indexing if true
  pdfFile?: string;
  pdfFiles?: { label: string; file: string }[];
}
```

**Step 2: Add `disabled: true` to all secondary sources in SOURCE_CONFIGS:**
Disable these 16 entries (add `disabled: true` to each):
- cgst-amend-2018, cgst-amend-2023, igst-amend-2018, cgst-jk-2017 (GST amendments)
- utgst-2017, utgst-amend-2018 (UTGST)
- sgst-delhi, sgst-haryana, sgst-himachal, sgst-madhya, sgst-punjab, sgst-jk (SGST state acts)
- fa-2019, fa-2020, fa-2021, fa-2022, fa-2023 (Finance Acts)

This reduces active sources from 23 to 6 core sources: comparison, act-2025, act-1961, cgst-2017, igst-2017, reference. Keep disabled entries in the array (do not remove them) so they can be re-enabled in future versions if needed.

**Step 3: Skip disabled sources in initRAG():**
In the `for (const cfg of SOURCE_CONFIGS)` loop inside initRAG(), add early skip:
```typescript
if (cfg.disabled) {
  console.log(`[RAG] Skipping disabled source: ${cfg.id}`);
  continue;
}
```

**Step 4: Exclude disabled sources from bucket balancing in retrieve():**
In retrieve(), where buckets are initialized (`for (const cfg of SOURCE_CONFIGS)`), skip disabled sources:
```typescript
for (const cfg of SOURCE_CONFIGS) {
  if (!cfg.disabled) {
    buckets.set(cfg.id, []);
  }
}
```
This prevents disabled sources from getting guaranteed bucket slots even if stale chunks somehow remain in the index.

**Step 5: Tune boost factors:**
- Comparison Guide: keep at 1.5 (proven effective from Phase 7, re-verify in golden query results)
- Tax Reference Guide: raise from 1.3 to 1.5 initially. After disabling secondary sources, re-run validate-reference.ts. If the 2 CII checks now pass, keep 1.5. If still failing, try 1.6.
- CGST Act 2017: no boost (keep as-is)
- IGST Act 2017: no boost (keep as-is)
- IT Act 2025: no boost (keep as-is)
- IT Act 1961: no boost (keep as-is)

**Step 6: If CII still fails after boost 1.5 and source reduction (CONDITIONAL — only if needed):**
Add length normalization to scoreChunk() ONLY for reference-source chunks. Apply after raw token frequency scoring, before boost multiplication:
```typescript
// Length normalization for reference chunks — prevents dense sections
// (calendar with ~40 date occurrences) from outscoring targeted sections (CII)
if (chunk.source === 'reference' && score > 0) {
  const lengthNorm = Math.sqrt(chunk.text.length);
  score = Math.ceil((score / lengthNorm) * 100);
}
```
Per CONTEXT.md: implement length normalization ONLY if golden query results show dense text crowding out relevant shorter chunks. If boost tuning alone fixes CII, skip this step entirely and note in SUMMARY.md that length normalization was not needed.

**Step 7: Verify the Comparison Guide boost level:**
After all changes, if validate-golden.ts shows Comparison Guide dominating ALL query types (GST + IT + Reference), reduce Comparison boost from 1.5 to 1.3 or 1.2. This is unlikely after secondary sources are removed but must be checked.
