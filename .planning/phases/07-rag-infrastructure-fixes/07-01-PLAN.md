<objective>
Refactor the RAG system from hardcoded 3-source architecture to a config-driven, source-agnostic design.

Purpose: The current retrieve() has three hardcoded bucket variables (fromComparison, from2025, from1961) and Chunk.source is a 3-value union — adding a 4th source (Phase 8 GST Acts) would require modifying retrieval logic, scoring, labeling, and initialization. This plan makes all four sites data-driven from a single SOURCE_CONFIGS array.

Output: Refactored server/rag/index.ts with SourceConfig registry, source-agnostic retrieval loop, stable chunk ID Map, topK=5 default, and detailed startup logging.
</objective>
