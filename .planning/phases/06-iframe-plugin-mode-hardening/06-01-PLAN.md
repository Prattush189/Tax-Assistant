Implement production-grade postMessage infrastructure in the iframe plugin mode. This plan covers three of the four PLUG requirements in a single commit: height reporting (PLUG-01), origin-validated inbound message handling (PLUG-02), and theme sync (PLUG-04). The server CSP is tightened as part of PLUG-02 (server side).

</objective>

    **useTheme.ts** — Add `setIsDarkMode` to the return object. The existing `useState` setter is already named `setIsDarkMode`; just add it to the return. Do NOT rename anything or touch the `useEffect` that syncs to localStorage and `document.documentElement`. New return: `{ isDarkMode, setIsDarkMode, toggleTheme }`.

    **usePluginMode.ts** — Rewrite to accept an optional `onSetTheme?: (dark: boolean) => void` parameter. Keep the existing `isPluginMode` useMemo exactly as-is. Add two `useEffect` hooks below it:

    Effect 1 — PLUG-01 height reporter + PLUG-02 ready signal:
    - Only runs when `isPluginMode` is true.
    - Defines `PARENT_ORIGIN = 'https://ai.smartbizin.com'`.
    - On mount: sends `window.parent.postMessage({ type: 'TAX_ASSISTANT_READY' }, PARENT_ORIGIN)` immediately (before ResizeObserver fires) so the parent can confirm the iframe is alive.
    - Creates `ResizeObserver` that observes `document.body`. In the callback: for each entry, computes `Math.ceil(entry.contentRect.height)` and posts `{ type: 'TAX_ASSISTANT_HEIGHT', payload: { height } }` to `PARENT_ORIGIN` (never `'*'`).
    - Returns cleanup: `resizeObserver.disconnect()`.
    - Dependency array: `[isPluginMode]`.

    Effect 2 — PLUG-02 origin validation + PLUG-04 theme sync:
    - Only runs when `isPluginMode` is true.
    - Defines `ALLOWED_ORIGINS = ['https://ai.smartbizin.com']` (array literal for strict equality checks via `.includes()`).
    - Handler: `(event: MessageEvent) => { if (!ALLOWED_ORIGINS.includes(event.origin)) return; if (event.data?.type === 'SET_THEME' && onSetTheme) { onSetTheme(event.data.dark === true); } }`.
    - Registers handler: `window.addEventListener('message', handler)`.
    - Returns cleanup: `window.removeEventListener('message', handler)`.
    - Dependency array: `[isPluginMode, onSetTheme]`.

    Hook returns `{ isPluginMode }` (unchanged API for non-plugin callers).

    Add these imports at top: `import { useMemo, useEffect } from 'react';` (replace existing `useMemo`-only import).

    **App.tsx** — In the App function, change:
    ```
    const { isDarkMode, toggleTheme } = useTheme();
    const { isPluginMode } = usePluginMode();
    ```
    to:
    ```
    const { isDarkMode, toggleTheme, setIsDarkMode } = useTheme();
    const { isPluginMode } = usePluginMode(setIsDarkMode);
    ```
    No other changes to App.tsx. The `setIsDarkMode` reference is stable (React useState setter identity is stable across renders — do NOT wrap in useCallback before passing).

    PITFALL to avoid: Do NOT use `postMessage(data, '*')` as targetOrigin anywhere. Always use `PARENT_ORIGIN` as the second argument. A wildcard allows any embedding page to intercept height messages.

    PITFALL to avoid: Do NOT duplicate theme state in usePluginMode. The setter from useTheme is the single source of truth. usePluginMode only calls the setter it receives.
