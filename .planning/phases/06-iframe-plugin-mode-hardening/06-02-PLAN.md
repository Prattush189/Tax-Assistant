  </files>
  <action>
    Read each file and identify elements likely to overflow at 400px. Apply targeted fixes. Do NOT restructure layouts or change functionality — only add/adjust Tailwind overflow classes.

    **Audit checklist per file:**

    **RegimeComparison.tsx** — This renders a side-by-side table comparing old vs new regime slabs. At 400px, two columns will likely overflow. Wrap the table container (the outermost div containing both regime columns) with `overflow-x-auto`. The wrapper should be a `div` with `className="overflow-x-auto"` surrounding the grid/table. This is the highest-risk component for horizontal overflow.

    **IncomeTaxTab.tsx** — Check for any grid layouts with min-width columns. If the input fields grid uses `grid-cols-2` without `sm:` prefix, it will stack too aggressively on narrow screens. Change any bare `grid-cols-2` to `grid-cols-1 sm:grid-cols-2` to stack fields on narrow widths. If already using responsive classes, no change needed.

    **CapitalGainsTab.tsx** — Same audit as IncomeTaxTab. Check for `grid-cols-2` without responsive prefix. Wrap result display tables in `overflow-x-auto` if they have fixed-width columns.

    **GstTab.tsx** — Check for horizontal layouts with fixed widths. The CGST+SGST vs IGST split display may have a two-column layout — wrap it in `overflow-x-auto` if the columns have fixed widths.

    **CalculatorView.tsx** — The tab shell itself (the tab buttons row for Income Tax / Capital Gains / GST). If the three tab buttons use a flex row that doesn't wrap, add `flex-wrap` or ensure tab labels are short enough to fit at 400px. If labels truncate awkwardly, add `text-sm` or shorten label text.

    **DashboardView.tsx** — The summary section may use a `grid-cols-3` or `grid-cols-4` for TaxSummaryCards. Change any bare `grid-cols-N` (N > 1) to use responsive prefixes: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` pattern. Check TaxWaterfallChart wrapper — if it has a `min-w-[Xpx]` or fixed width, wrap the chart container in `overflow-x-auto`.

    **TaxSummaryCards.tsx** — Check if the cards grid is defined here. If so, apply the same `grid-cols-1 sm:grid-cols-2` responsive fix.

    **Principle:** If a container's content can be wider than 400px, wrap it in `overflow-x-auto` so content is scrollable rather than clipped or overflowing the page. If a grid can stack, use responsive `grid-cols-1 sm:grid-cols-2` to stack at narrow widths.

    **Do NOT** add `isPluginMode` conditional to these fixes — constrained-width responsive behavior benefits all users, not just plugin mode.

    After edits, run TypeScript check to confirm no type errors introduced by JSX changes.
