Introduce TaxCalculatorContext to share income tax state between CalculatorView and DashboardView.

Purpose: The dashboard needs IncomeTaxResult data that currently lives in IncomeTaxTab local state. Lifting to context is the correct architectural solution — it avoids prop-drilling through App.tsx (which must stay a thin 58-line shell), avoids duplicating state, and keeps both views reactive to the same inputs.

- `src/contexts/TaxCalculatorContext.tsx` — provider + hook
- `src/components/calculator/IncomeTaxTab.tsx` — migrated to use context (same UI behavior)
- `src/App.tsx` — TaxCalculatorProvider wraps <main> so both CalculatorView and DashboardView are inside the provider
</objective>
